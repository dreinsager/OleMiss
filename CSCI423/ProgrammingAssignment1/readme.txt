Part A.c will output the results on the screen as to what wil appear in each file. It then takes the info from within input.txt to the other file output.txt

Part B.c will do same as Part A but will return the output result in a differentorder. (i.e. input.txt will be 4 3 2 1 output.txt will store 1 2 3 4)

To compile: gcc PartA.c or gcc PartB.c

To run: ./a.out PartA.c <filename> <filename> OR ./a.out PartB.c <filename> <filename>
